使用Hugo+GitHub Pages 搭建个人博客

2020年2月27日
| 教程
| 阅读
文章目录

Hugo是什么

Hugo是一个静态站点生成工具。可以根据Markdown文件生成静态站点文件。 同类型的产品还有Hexo、JEKYLL、Next.js等。

下载安装Hugo

Windows可根据官网教程，到Hugo release中下载最新的可执行文件。 下载解压后，将解压路径添加进环境变量中，命令行中执行 hugo version 能够正常显示即可。

如何使用Hugo

步骤一：创建一个博客站点的框架,并进入到站点根目录下

hugo new site /path/to/site #你的站点路径
cd /path/to/site #进入站点根目录

步骤二：到官网提供的主题列表中选择合适的主题安装。

git clone https://github.com/flysnow-org/maupassant-hugo themes/maupassant
# 将示例中配置文件复制到站点根目录
cp themes/maupassant/exampleSite/config.toml config.toml

PS. 我个人认为很好看的主题

Beautifulhugo : 清爽好看

DGraph：搭配简单

步骤三：创建文章内容页面

hugo new /posts/my-first-post.md

生成文件如下

---
title: "my-first-post"
date: 2020-02-28T08:41:53+08:00
draft: true  # 是否为草稿，如果是草稿则不会打包成静态站点
---

步骤四：本地执行Hugo Server 查看文章草稿样式

hugo server -D

步骤五： 打包，将Markdown文件转换成HTML文件

首先需要my-first-post.md文章上面的draft参数改为false或者直接删除

hugo -t  maupassant

至此Hugo相关的基础用法就介绍到这里。可以看出来Hugo实际上最主要的作用将Markdown文件转换成HTML文件的作用。 同时，可以进行一些静态站点的配置，当然可以通过一些插件来丰富Hugo的功能。

GitHub Pages是什么

Github Pages可以理解成为一个静态资源托管服务器。不过这个托管服务器直接指向Github Repository。 官网介绍。

GitHub Pages is a static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub, optionally runs the files through a build process, and publishes a website.

这样带来的好处就是，通过GitHub pages管理的站点，可以通过Git进行文件的管理,同时只要Github上 的内容出现更改，站点可以同步进行更新。因此我们只要依靠Github Pages，无需服务器就可以搭建出以静态页为主的个人博客。

部署Github Pages

站点经过打包之后就是纯粹的HTML文件，将public文件下的代码提交到个人帐号命名的xxx.github.ioRepository上。 访问https://xxx.github.io就可以看到生成的静态网页内容了。

最后，我们也可以按照Hugo官方推荐的脚本，设置一下自动化打包部署到Github Pages上去。

# 删除打包文件夹
#rm -rf public

总结

整个搭建博客的过程还是很简单的，不需要处理服务器相关的事情，可以更多的把精力放在写作上面。博客的访问速度、安全性都相对更好一些。 当然，因为使用的静态站点生成工具Hugo，所以没有办法很好的支持一些动态的内容。 可以通过对接第三方扩展实现功能。 总的来说，这套方案很适合不想多折腾，专注于写博客的人。
